The Gap

Agents think. Robots move.
Neither does both.

Large language models can decompose complex goals into steps. Robots can execute precise motor control. But there's no standard way to connect intent to physical execution. Skills are hardcoded, brittle, and locked inside proprietary stacks.

CLAWS defines an open skill layer — a shared vocabulary between agents and the physical world.

# agent decomposes a goal
agent> deliver package to room 204

# CLAWS resolves to physical skills
claws> navigate_to(building_b, floor_2)
claws> open_door(room_204)
claws> place_object(package, desk)

# execution feedback
[ok] navigated — 47s, 0 replans
[ok] door opened — handle type: lever
[ok] placed — clearance verified
Skill Domains

What agents can do in the physical world

Each skill is a composable, observable unit of physical capability that an agent can invoke, monitor, and recover from.

Navigation

Autonomous movement through mapped and unmapped environments. Waypoint sequencing, obstacle avoidance, dynamic replanning.

Manipulation

Pick, place, push, pull, insert. Grasp planning with force feedback and object-aware contact strategies.

Perception

Scene understanding, object detection, spatial reasoning. Feed structured observations back to the agent for decision-making.

Interaction

Doors, elevators, switches, touchscreens. Physical interface skills that let robots operate in human-designed spaces.

Safety & Recovery

Collision detection, graceful degradation, human handoff. Every skill reports its confidence and knows when to stop.

Composition

Chain skills into multi-step behaviors. Agents orchestrate sequences; CLAWS handles timing, dependencies, and error propagation.
